```python
import numpy as np

def select_next_node_v2(current_node: int, destination_node: int, unvisited_nodes: set, distance_matrix: np.ndarray) -> int:
    """
    Select the next node to visit from the unvisited nodes using a look-ahead heuristic.
    """
    # Look-ahead heuristic: choose the node that minimizes the sum of the current distance
    # and the estimated distance to the remaining nodes.
    scores = {}
    for node in unvisited_nodes:
        # Calculate the current distance from the current node to the potential next node.
        current_distance = distance_matrix[current_node][node]
        # Estimate the remaining distance from the potential next node to the destination.
        # We use the minimum distance to any of the remaining nodes as a heuristic.
        remaining_distance = min(distance_matrix[node][i] for i in unvisited_nodes if i != node)
        # Calculate the score as a sum of the current distance and the estimated remaining distance.
        scores[node] = current_distance + remaining_distance

    # Find the node with the minimum score (the shortest total estimated path).
    next_node = min(scores, key=scores.get)
    return next_node
```
This function uses a simple look-ahead heuristic that sums the current distance to a node with the minimum distance from that node to any of the remaining unvisited nodes. The node with the lowest sum is chosen as the next node to visit. This heuristic does not take into account the potential for sub-tour formation, but it is a simple and effective way to reduce the search space for the TSP problem.
